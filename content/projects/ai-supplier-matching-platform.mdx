## Project overview

A full stack platform concept designed to help businesses identify suitable suppliers for project materials using structured inputs and matching logic.

## Problem

Businesses often struggle to find reliable suppliers quickly, especially when requirements are complex or time sensitive.

## Context

This work was completed with a startup team, where requirements evolved rapidly and the focus was on testing the product idea quickly.

## My role

Full stack development support and architectural contribution, including front end features, API integration, and data modelling.

## Goals

- Capture structured project requirements without friction
- Return relevant supplier matches quickly
- Build a data model that can scale as more suppliers are added

## Technical architecture

- React front end for requirement capture and results
- C# API services for matching logic and data access
- PostgreSQL for supplier and project data
- Azure for hosting and service integration

## Implementation decisions

- Structured input forms to improve data consistency
- Rule based matching for initial accuracy and transparency
- Modular API endpoints to allow future ML scoring

## Trade offs

- Prioritised explainable matching over complex ML ranking in early stages
- Limited advanced filtering in order to ship a working core faster

## Challenges

- Designing a flexible schema for varied supplier attributes
- Keeping matching logic fast as data grows

## Solutions

- Normalised data model with indexed search fields
- Clear separation of matching rules from UI presentation

## Lessons learned

- Data structure decisions made early define future matching quality
- Shipping a clear baseline is better than waiting for perfect automation
