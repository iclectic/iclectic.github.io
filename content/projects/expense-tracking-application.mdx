## Project overview

A Flutter mobile application designed to help users track spending and visualise cumulative totals over time.

## Problem

Many people track expenses across multiple tools or rely on manual spreadsheets, which makes trends hard to see.

## Context

This project explored how a clean mobile interface combined with data visualisation can support better financial awareness.

## My role

I designed the user experience and built the application end to end in Flutter.

## Goals

- Make expense entry fast and low friction
- Present cumulative totals clearly in a single view
- Ensure the UI remains calm and readable over long periods

## Technical architecture

- Flutter client with local storage
- Data aggregation layer for cumulative totals
- Charting library for visualisation

## Implementation decisions

- Local storage to keep the app responsive and private
- Simple chart types to reduce visual noise
- Focus on cumulative totals rather than category breakdowns for v1

## Trade offs

- No cloud sync in the initial release to keep the scope tight
- Limited category features to prioritise clarity over complexity

## Challenges

- Maintaining chart readability as data grows
- Preventing input friction during daily use

## Solutions

- Smoothed chart display and simple colour palette
- Minimal input form with defaults and quick edits

## Lessons learned

- Clear visual hierarchy matters more than feature volume in personal finance tools
- Small daily interactions shape overall retention
